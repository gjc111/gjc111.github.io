title: MFC（基于对话框）学习总结

MFC（基于对话框）学习总结

#环境：VS2019

软件课设，要求做一个医院病人监护系统，小组三人，老师建议C或者C++，我负责GUI模块，摸索了很久选择了用C++，学习了MFC库，做一个小的总结在这里

一，创建工程文件

创建MFC应用程序，选择应用程序类型为基于对话框

二，基本控件学习及基本使用

1，按键（button）

从工具箱拖入到Dialog框中,双击按钮跳转到进行点击按钮的响应代码编写

同时学习了messagebox（TEXT（“内容”））；

此语句可以弹出小框显示出内容。

2，静态对话框(Static Text)

单击后可以编写汉字静态持续显示

3,编辑栏（Edit Control）（注：需要关联变量）

可以输入内容，关联变量（值类型）后打开update（）其他程序中引用此变量为编辑框内的内容

变量有值类型和控件类型，其中值类型可以引用变量表示输入内容

特别当关联变量后，如果需要更改，需要在变量管理内删除原有变量，再进入.cpp文件内注释原有变量声明（不会自动删除）

4，列表（List Control）（注：需要关联变量）

相关语句

（lista为关联变量名，自行设定）

1.lista.InsertColumn(序号（0开始）, “插入内容”, LVCFMT_LEFT（左对齐）, 50（位置）);

插入表头即第一行头

2.lista.InsertItem(序号n（0开始）, TEXT(“内容”));插入第n行的第1格的内容

3.lista.SetItemText(行序号（0开始）, 列序号（1开始）, TEXT(“内容”));

前两步骤初始化列表行头和列头 第三步用来填充内部空格内容

5，其他界面连接（注：需要在子界面添加类）

资源视图中Dialog文件夹右击插入Dialog，点击进入子界面进行规划后，右击界面添加类，取类名，

主界面使用前主界面的.cpp文件头文件先include”类文件名.h“，之后进行定义对象后使用。

常用语句

对象名.domodal（）；跳出子界面框框

三，疑难点处理

1.MFC特有类型Cstring类型

Cstring类型中存储的是宽字节字符串，一般的通信中多使用多字节字符串，需要相互转化后使用，这一部分在软件课设过程中处理了很久，找了很多办法才实现

实现代码：

DWORD num = WideCharToMultiByte(CP_ACP, 0, A1, -1, NULL, 0, NULL, 0);A1为宽字节字符数组名

//开辟空间
char* tChar = NULL;
tChar = (char*)malloc(num * sizeof(char));
if (tChar == NULL)
{
    free(tChar);
}
memset(tChar, 0, num * sizeof(char));

//将宽字节字符串转换为多字节字符串
WideCharToMultiByte(CP_ACP, 0, A1, -1, tChar, num, NULL, 0);

tChar为多常用的一维字符指针，最终保存得到的转化后的结果。

2.不同界面内部变量值的传递

MFC很奇怪，正常C++的设置多个.cpp内可以公用的变量方法会报错，解决不了，查了很多资料，发现得用这个办法在多个.cpp文件内传递。

在任意一个源文件内部

写子类（子类名xxx）继承自CWinApp类，在其中定义

public:
变量名；

​ …

.cpp文件内include之前的.h文件

xxx* APP = (xxx*)AfxGetApp();

创建类指针，然后APP->变量名实现多个.cpp文件内的公用。

四，其他内容学习

多线程

CreateThread(NULL, NULL, (LPTHREAD_START_ROUTINE)子程序名称,NULL, NULL, NULL);

在此处分两条路并行，主线程继续向下执行，子线程向子程序那边进行向下执行。
